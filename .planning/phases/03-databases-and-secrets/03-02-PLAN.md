<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| var.master_password → aws_db_instance.tenant | Sensitive password (from ssm module) crosses into RDS instance password argument — stored encrypted in S3 state |
| aws_security_group.rds_tenant ingress | Port 5432 must only be reachable from the task SG; any CIDR-based ingress here would expose the database to the internet via public subnets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Information Disclosure | modules/rds-tenant/main.tf — master_password in state | mitigate | password = var.master_password (received as sensitive variable); lifecycle { ignore_changes = [password] } prevents re-exposure on plan diff; state bucket encrypted (AES-256 SSE per bootstrap) |
| T-03-05 | Elevation of Privilege | aws_security_group.rds_tenant ingress | mitigate | ingress rule uses security_groups = [var.task_security_group_id] only; no cidr_blocks on 5432 rule; ASVS L1 requirement; verify is a concrete grep assertion |
| T-03-06 | Information Disclosure | publicly_accessible = false | mitigate | RDS instance not directly reachable from internet; combined with SG-reference ingress, attack surface is limited to tasks in the task SG |
| T-03-07 | Tampering | deletion_protection + skip_final_snapshot | mitigate | deletion_protection = true prevents accidental destroy; skip_final_snapshot = false ensures data snapshot on any destroy attempt (D-08) |
</threat_model>
