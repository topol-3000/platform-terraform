<objective>
Implement modules/ssm: generate RDS master passwords and HMAC salt via random_password, write each to an aws_ssm_parameter SecureString, and export sensitive raw values for inter-module wiring plus non-sensitive names/ARNs for the provisioner contract.

Purpose: SSM is the single authoritative secret generator. It must be implemented first (Wave 1) because rds-tenant and rds-control-plane consume its sensitive password outputs. Implementing SSM before the RDS modules avoids a forward-reference bootstrapping problem and keeps the secret generation chain centralised.

Output: modules/ssm/{variables,main,outputs}.tf fully implemented. No envs/prod wiring yet — that lands in Plan 05.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| random_password → aws_ssm_parameter | Generated secret crosses from Terraform state into AWS SSM SecureString — encrypted at rest by alias/aws/ssm |
| ssm outputs → rds-tenant/rds-control-plane modules | Sensitive raw password value passes between Terraform modules in-memory at plan/apply time; never written to a non-sensitive output |
| envs/prod/outputs.tf → provisioner adapter | Only non-sensitive SSM parameter names/ARNs cross this boundary; raw secret values must never appear here |
