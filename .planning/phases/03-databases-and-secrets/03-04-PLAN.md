<objective>
Implement modules/rds-proxy: pre-declare all required variables (D-11), then implement the full proxy resource set — all 8 resources gated behind count = var.enable_rds_proxy ? 1 : 0 (default false) — and export the endpoint with a try() guard so envs/prod wiring stays valid when the flag is off.

Purpose: RDS Proxy reduces connection exhaustion at ~30 active tenants. The code is fully present today but zero proxy resources appear in the default plan. When the flag is flipped, a single tfvars change activates the proxy without code changes. The try() guard on the endpoint output is mandatory — without it, envs/prod wiring fails at plan time when count = 0. This plan depends on Plan 02 (rds-tenant must be declared before its identifier can be referenced here).

Output: modules/rds-proxy/{variables,main,outputs}.tf fully implemented. Plan 05 (envs/prod wiring) will pass module.rds_tenant.identifier as db_instance_identifier and the enable_rds_proxy flag.
</objective>
