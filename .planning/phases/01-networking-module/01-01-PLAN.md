<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet → ALB SG | Untrusted public traffic enters on 80/443 from 0.0.0.0/0. |
| ALB SG → task SG | Only the ALB may reach tenant tasks on 8069 (SG-reference, not CIDR). |
| public subnet → internet | Tasks have public IPs (no NAT); egress is allow-all this milestone. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure / Elevation | aws_security_group.task port 8069 | mitigate | Ingress uses `security_groups = [aws_security_group.alb.id]` (not a CIDR), so public-subnet tasks are reachable on 8069 only via the ALB — locked decision D-09/NET-04, asserted in Task 2 acceptance criteria. |
| T-01-02 | Spoofing / DoS | aws_security_group.alb 80/443 | accept | Public web entry on 80/443 from 0.0.0.0/0 is the intended ALB surface; rate-limiting/WAF is out of scope for the networking milestone. |
| T-01-03 | Information Disclosure | egress allow-all on task SG | accept | Allow-all egress accepted this milestone (tasks need outbound to ECR/SSM/internet with no NAT); egress tightening explicitly deferred (CONTEXT Deferred Ideas). |
| T-01-04 | Information Disclosure | outputs.tf | mitigate | Outputs are resource ids/CIDRs only — no secrets, credentials, or plaintext sensitive data leave the module (CLAUDE.md anti-pattern guard). |
| T-01-SC | Tampering | terraform provider/module installs | mitigate | No new packages installed by this plan (modules use the registry-resolved hashicorp/aws provider already constrained in envs/prod/versions.tf); no npm/pip/cargo installs. |
</threat_model>
