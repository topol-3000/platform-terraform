<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HCL variable input → module | var.tenant_domain flows in from envs/prod; validation block is the guard |
| Terraform plan → module resources | Offline gate; no real AWS calls; no runtime attack surface this phase |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | modules/acm variable tenant_domain | mitigate | validation block with can(regex(...)) rejects empty string and obviously-invalid domains before any resource is planned; satisfies ASVS V5 input validation at the IaC layer |
| T-05-02 | Tampering | modules/route53 variable tenant_domain | mitigate | identical validation block as T-05-01; same guard in both modules so that either module alone catches an invalid domain |
| T-05-03 | Info Disclosure | modules/acm outputs | accept | cert_arn is a non-secret identifier (ARN); no credentials, no private key material in outputs or state; ASVS V6 Cryptography: ACM manages cert lifecycle and private key, never exposed to Terraform |
| T-05-04 | Spoofing | Public Route53 zone (DNS hijack) | accept | Zone is not created until real terraform apply; this phase is plan-only/offline. At real apply time, NS records are AWS-managed; per-tenant records are adapter-controlled. Low-value target without applied infrastructure. |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan; all resources use the already-pinned hashicorp/aws 6.51.0 provider |
</threat_model>
